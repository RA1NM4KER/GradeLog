# Security Policy

## Supported Versions

Security fixes are only provided for the latest version on the `main` branch.

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately by email:

`kefasa112@gmail.com`

Please do not open a public GitHub issue for security reports.

When reporting a vulnerability, include:

- a clear description of the issue
- steps to reproduce it
- the affected area or file, if known
- any proof of concept or screenshots that help explain the risk

I maintain this project solo, so response times may vary, but I will review legitimate reports as soon as possible.

Please give me reasonable time to investigate and address the issue before making any public disclosure.
